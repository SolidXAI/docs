We can elaborate a little bit more on models and fields, because in SolidX models are able to express semantic relationships, hold more metadata like which is the data source a model is able read / write its data from, does a model implement soft delete functinality, does a model support internationalization etc... 

Similarly fields exhibit more semantic types like there are fields of type one2many, many2one, many2many, selectionStatic (allows creating a drop-down using a static list of values), selectionDynamic (allows creating a drop-down using dynamic data sources like 3rd party APIs to fetch list of countries or currencies), mediaSingle (allows one to upload a single attachment could be of any type - pdf, image, video, audio) or mediaMultiple (allows one to upload multiple media files). 

These are a few examples there are more but want to give a flavour to the user reading this in the intro itself.
