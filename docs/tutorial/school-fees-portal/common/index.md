# Common Reference

This section contains documentation that is shared across multiple use cases in the School Fees Portal tutorial. Instead of repeating the same information in each use case, we've consolidated it here for easy reference and maintenance.

## Available References

### [Field Type Reference](./field-types.md)
A comprehensive guide to all field types available in SolidX when creating data models. Use this as a quick reference when building your models.

### [Important Configuration Notes](./configuration-notes.md)
Essential configuration guidelines for relations, cascading, audit tracking, indexes, and other important settings when creating models.

### [Generating APIs and UI Components](./code-generation.md)
Step-by-step instructions for using SolidX's App Builder to auto-generate REST APIs and UI components from your data models.

### [Applying View Customizations](./applying-view-customizations.md)
Step-by-step instructions for applying customized form and list view layout JSON through the Layout Builder.

## When to Reference These Docs

You'll find links to these common reference documents throughout the use case documentation, particularly in the "Building the Data Models" and "Implementing Custom Business Logic" sections. Whenever you see a reference like:

> 📚 **Common Reference:** For field type details, see [Field Type Reference](../common/field-types.md)

You can click through to get detailed information without having to search through the entire documentation.
