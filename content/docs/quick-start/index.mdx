## Prerequisites

| Requirement | Version | Required for | Check |
|---|---|---|---|
| **Node.js** | 22+ (LTS recommended) | All modes | `node -v` |
| **Docker** | Latest stable | Default manual path - spin up PostgreSQL locally if you do not already have an instance | `docker --version` |
| **PostgreSQL** | 14+ | Default manual path - connect to an existing instance, or use Docker to start one | `psql --version` |
| **Python** | 3.11+ | Agent & MCP only | `python3 --version` |

Node.js is the only hard requirement for all paths. The default PostgreSQL path in Step 2 requires Docker or an existing PostgreSQL instance. The optional embedded path in Step 2 needs nothing else. Python is only needed if you plan to use the [SolidX AI Agent](/docs/quick-start/using-solidx-ai-agent) or [MCP server](/docs/quick-start/using-solidx-mcp-server).

<Callout type="warning" title="Full PostgreSQL vs. embedded - when to use which">

The default **full PostgreSQL path** (Path A in Step 2) is recommended for real development and **any production usage**.

The optional **embedded PGlite path** (Path B in Step 2) is the fastest way to **try out SolidX** - no Docker, no PostgreSQL install, no credentials. It is ideal for evaluation, demos, and local experimentation. PGlite is a single-process in-process engine and is not intended for production workloads, concurrent access from multiple services, or large datasets.

</Callout>

## Mode 1: Manual

### Step 1: Install solidctl

Install the SolidX CLI globally so it's available from any directory:

```bash
npm i -g @solidxai/solidctl@latest
```

Verify the installation:

```bash
solidctl --version
```

### Step 2: Scaffold the project

#### Path A: Full PostgreSQL server (default - recommended for production)

You can connect to an existing PostgreSQL instance or spin one up with Docker. A full PostgreSQL server is recommended for **any production usage**.

If you don't already have PostgreSQL running, start a container:

```bash
docker run -d \
  --name SolidX_DB \
  -e POSTGRES_USER=solidx_app_user \
  -e POSTGRES_PASSWORD=strongpassword \
  -e POSTGRES_DB=solidx_app_db \
  -p 5432:5432 \
  -v solidx_pgdata:/var/lib/postgresql/data \
  postgres:17
```

Then scaffold against it:

```bash
solidctl create-app \
  --name my-solid-app \
  --no-interactive \
  --db-client PostgreSQL \
  --db-host localhost \
  --db-port 5432 \
  --db-name solidx_app_db \
  --db-username solidx_app_user \
  --db-password strongpassword
```

<Callout type="warning" title="Change the default credentials">

The username (`solidx_app_user`), password (`strongpassword`), and database name (`solidx_app_db`) shown above are for a fast first run only. **Change them to values you control** before any real use, and use those same values in the `create-app` command above.

</Callout>

If you have an existing PostgreSQL database, replace the `--db-*` values with your own credentials. Or run the interactive scaffolding and enter your credentials when prompted:

```bash
solidctl create-app
```

<details>
<summary><strong>Path B: Embedded database (optional - fastest way to try SolidX)</strong></summary>

Scaffold a project with the embedded PGlite database in a single one-shot command:

```bash
solidctl create-app --no-interactive --embedded --name my-solid-app
```

No Docker, no PostgreSQL, and no credentials required. `solidctl` configures and manages the embedded database for you automatically during `create-app`, `build`, `seed`, and `start:dev`.

<Callout type="info" title="Agent and MCP support in embedded mode">

The [SolidX AI Agent](/docs/quick-start/using-solidx-ai-agent) and [MCP server](/docs/quick-start/using-solidx-mcp-server) are in **early alpha** with embedded mode at the moment. For agent or MCP workflows, use the default full PostgreSQL path (Path A) above.

</Callout>

</details>

<Callout type="info" title="Info">

The `create-app` command installs all npm dependencies for both backend and frontend. This is a one-time cost; subsequent commands like `build` and `seed` are much faster.

</Callout>

### Step 3: Start the services

Move into the project folder and start both the backend API and frontend admin panel in one supervisor:

```bash
cd my-solid-app && solidctl start:dev
```

Open the admin panel at `http://localhost:3001` and log in:

| Field | Value |
|---|---|
| Username | `sa` |
| Password | `Admin@3214$` |

That's it. Your SolidX project is running. Start building modules via the admin panel, or extend the generated code directly.

## Mode 2: Agentic

Paste the one-line prompt below into your AI coding assistant (Claude Code, Cursor Agent, Codex CLI, Windsurf, etc.) from the directory where you want the project created.

<Callout type="info" title="Info">

Agentic mode scaffolds against the same **full PostgreSQL** setup recommended for the manual path (spinning up a Docker container automatically, or connecting to an existing instance you point it at), then finishes the job for you: it runs `solidctl mcp install` inside the new project so your AI client's MCP config is wired up automatically, and starts the API, UI, and MCP server together with `solidctl start:dev`. The prompt points your agent at a self-contained instruction file that covers all of this, including a fallback to the embedded database if Docker and PostgreSQL aren't available.

</Callout>

```text
Set up a new SolidX project for me. Read and follow every step in https://docs.solidxai.com/agentic-install.md and give me a single report at the end.
```

Your AI agent will execute all the steps, including connecting itself to the SolidX MCP server, and report back once with everything you need when the project is ready to use.

<Callout type="warning" title="Restart your AI client after setup">

`solidctl mcp install` updates your AI client's MCP configuration, but a running session can't reload it on its own. Your agent's final report will include a command to start the project yourself in your own terminal - run that first, then restart the current Claude Code / Cursor / Codex session or window so it picks up the new SolidX MCP tools.

</Callout>

## Common solidctl Commands

| Command | What it does |
|---|---|
| `create-app` | Scaffold a new project |
| `build` | Build backend + frontend |
| `seed` | Load metadata and system user |
| `generate module` | Regenerate code from metadata |
| `upgrade` | Upgrade SolidX dependencies |
| `info --detailed` | Show project info |

Full reference: [solidctl Commands](/docs/developer-docs/solidctl-commands)
