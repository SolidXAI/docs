## Set Up the Project

<Callout type="info" title="Outcome">

At the end of this page, you should have:

- PostgreSQL running in Docker
- a scaffolded `todo-app` project
- the SolidX admin UI running on `http://localhost:3001`
- the API and Swagger docs running on port `3000`

</Callout>

## Quick Path

<Steps>
<Step>

### Start PostgreSQL

Bring up the external PostgreSQL container used by the tutorial.

</Step>
<Step>

### Scaffold the App

Run `solidctl create-app` and answer the scaffold prompts for the Todo project.

</Step>
<Step>

### Start SolidX

Run the generated app locally and verify the admin UI and Swagger docs are reachable.

</Step>
</Steps>

## Before You Start

Install these prerequisites:

- Docker Desktop or Docker Engine
- Node.js 22+
- `solidctl`
- Python 3.11+ if you plan to use the MCP flow

Verify the environment:

```bash
docker --version
solidctl --version
python3 --version
```

References:

- [SolidX Quick Start](https://docs.solidxai.com/docs/quick-start)
- [Using the SolidX MCP Server](https://docs.solidxai.com/docs/quick-start/using-solidx-mcp-server)

## Start PostgreSQL in Docker

This tutorial uses an external PostgreSQL container:

```bash
docker run -d \
  --name SolidX_DB \
  -e POSTGRES_USER=solidx_app_user \
  -e POSTGRES_PASSWORD=strongpassword \
  -e POSTGRES_DB=solidx_app_db \
  -p 5432:5432 \
  -v solidx_pgdata:/var/lib/postgresql/data \
  postgres:17
```

These values are fine for local tutorial use. Change them for any real environment.

## Scaffold the App

<Callout type="tip" title="Recommended order">

Start PostgreSQL first, then run `solidctl create-app`, then start the generated project with `solidctl start:dev`.

</Callout>

Run:

```bash
solidctl create-app
```

Scaffold the project:
![Project scaffold setup](/img/tutorial/todo-app/setup-project-scaffold.png)

## Start the App

```bash
cd todo-app
solidctl start:dev
```

Then sign in to `http://localhost:3001` with:

- Username: `sa`
- Password: `Admin@3214$`

Available endpoints:

- API: `http://localhost:3000`
- Swagger: `http://localhost:3000/docs`
- Admin UI: `http://localhost:3001`

<Callout type="info" title="Before moving on">

Only continue to the MCP step after you confirm the admin UI loads and you can sign in with the default super admin account.

</Callout>
