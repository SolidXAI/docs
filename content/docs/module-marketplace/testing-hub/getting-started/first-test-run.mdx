This walkthrough helps you take a product from no tests to a useful release-readiness view.

## 1. Add your product

Open **Applications** and select **New Application**. Add a clear name, such as *Library Management* or *School Fees Portal*.

Also add the product's addresses:

- **Frontend URL** — the web address people use to open the product.
- **Backend URL** — the web address of the product's service behind the screens.

Testing Hub uses these addresses to test the customer journey from the screen through to the product's service.

## 2. Create test scenarios with the MCP AI assistant

Think about the tasks customers or staff cannot do without. For a library product, that might be:

- A librarian can sign in.
- A librarian can add a book.
- A member can find a book.

Open the product's **Scenarios** tab and select **Author Scenarios**. In the connected MCP AI assistant, describe the journeys in plain language and ask it to create test scenarios for the product.

For example, ask: *“Create test scenarios for the Library Management product: a librarian can sign in, add a book, and find a book.”*

The MCP AI assistant creates the test scenarios from your description. You only need to explain the customer journey and expected outcome; you do not need to describe technical steps.

## 3. Run the test scenarios

Open your product and choose **Start Run**. Run all test scenarios for a full release review, or choose a label such as *Sign in* or *Payments* to test one area.

As the tests run, Testing Hub shows which journeys are waiting, in progress, successful, or need attention.

## 4. Read the result

When the run ends, look first at the overall result and then at any journeys that need attention. A successful run gives confidence that the selected journeys are working. A failed run tells you exactly which journey needs follow-up.

## 5. Share and act

If a test needs attention, share the failed journey with the delivery team. The result includes a replay and supporting details that help them understand the problem. When it is fixed, run the same tests again to confirm the product is ready.

Next, learn how to [schedule regular tests](/docs/module-marketplace/testing-hub/automation/schedules) so your team does not have to remember to run them manually.
