Start a test run when you want a current view of product health.

## Run the right tests

1. Open the product in **Applications**.
2. Select **Start Run**.
3. Run everything for a full release review, or choose a label to focus on one area, such as **Payments** or **Sign in**.
4. Select **Start run**.

## Watch progress

Testing Hub updates the result while tests are in progress. You can see which customer journeys are still waiting, being tested, and which have passed or need attention.

<Callout type="tip" title="Use focused tests after a change">
  If a change only affects payments, begin with the **Payments** group. Before releasing more broadly, run your full release-test group as well.
</Callout>
