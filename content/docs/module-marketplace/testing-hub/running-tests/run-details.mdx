Each test run gives you a summary first, followed by the result of every customer journey included.

## Start with the overall result

Look at these points first:

- **Overall result** — whether the selected journeys are ready or need attention.
- **Passed journeys** — the journeys that are working as expected.
- **Journeys needing attention** — the exact experiences that did not complete as expected.
- **Time taken** — useful when comparing the result with earlier runs.

## What to do when a journey needs attention

1. Open the failed journey to see its result.
2. Review the replay, if one is available.
3. Share the result with the delivery team responsible for that part of the product.
4. After the issue is fixed, run the same test again.

The history for each product helps you show whether a problem is new, resolved, or happening repeatedly.

<Callout type="info" title="A failed test is useful information">
  A failed test does not automatically mean the whole release must stop. Use the affected customer journey and the importance of the change to make the right product decision.
</Callout>
