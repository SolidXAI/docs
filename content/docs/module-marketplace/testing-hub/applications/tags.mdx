title: Group Important Tests
icon: "tag"
description: Use simple labels to organise customer journeys and run the tests that matter right now.
---


Labels help you organise tests into meaningful groups. They make it easy to focus on one part of the product instead of always running everything.

## Good ways to use labels

Use labels that reflect how your product team thinks about the work:

- **Payments** for journeys that collect or confirm payments.
- **Sign in** for access and account journeys.
- **Release test** for the small set of journeys required before every release.
- **Customer support** for journeys your support team relies on.

## Use labels when you run tests

When you start a test run, select a label to focus only on that group. For example, after changing the payment process, run **Payments** first. Before a full release, run **Release test** or all tests.

<Callout type="tip" title="Keep labels simple">
  Use a small, shared set of labels. Clear names are more useful than a long list that no one remembers to use.
</Callout>
