# Dashboarding

SolidX dashboarding is a **metadata-driven framework capability** for building operational and analytical dashboard experiences without hard-coding each screen.

It is designed around four principles:

- metadata-first dashboard definitions,
- backend provider-driven data resolution,
- frontend extension-driven widget rendering,
- and per-user layout personalization.

## Current Runtime Route and API Model

UI route:

- `/admin/core/:moduleName/dashboard/:dashboardName`

Runtime API surface:

- `GET /dashboard/:moduleName/:dashboardName/definition`
- `POST /dashboard/:moduleName/:dashboardName/data`
- `POST /dashboard/:moduleName/:dashboardName/widgets/:widgetName/data`
- `GET /dashboard/:moduleName/:dashboardName/variable-options/:variableName`
- `GET /dashboard/:moduleName/:dashboardName/layout`
- `PUT /dashboard/:moduleName/:dashboardName/layout`

All runtime dashboard APIs are JWT-protected.

<Callout type="info" title="Source of Truth">

Dashboard definitions are resolved from module metadata JSON (`dashboards` section).
Only user-specific layout overrides are persisted in the database.

</Callout>
