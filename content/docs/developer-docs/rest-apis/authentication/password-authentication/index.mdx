## Password Authentication

This section covers the password-based authentication APIs available in SolidX.

<Callout type="info" title="Mental Model">

Password authentication establishes a SolidX session from user credentials and then keeps that session active through access and refresh tokens.

- Registration creates the user account when public registration is enabled.
- Authentication exchanges credentials for tokens.
- Refresh, password reset, user info, and logout manage the lifecycle of that session.

</Callout>

## Implementation Overview

SolidX provides a comprehensive password-based authentication mechanism with the following endpoints:

1. **Register**
2. **Authenticate**
3. **Refresh Tokens**
4. **Forgot Password**
5. **Change Password**
6. **Get User Info**
7. **Logout**

<Callout type="info" title="Token Lifecycle">

The authenticate endpoint returns access and refresh tokens. Use the access token for protected API calls, and use the refresh token when the access token expires.

</Callout>

## 1. Register

Allows users to create a new account.

### Environment Variables

- `IAM_PASSWORD_REGISTRATION_ENABLED`: Enables/disables registration.
- `IAM_ALLOW_PUBLIC_REGISTRATION`: Allows public registration when set to `true`.

<Tabs defaultValue="request">
  <TabItem value="request" label="Request">

  ### Headers

  ```http
  Content-Type: application/json
  ```

  ### Request Body

  ```json
  {
    "fullName": "string",
    "username": "string",
    "email": "[EMAIL]",
    "password": "[PASSWORD]",
    "mobile": "string",
    "roles": ["string"]
  }
  ```

  </TabItem>
  <TabItem value="response" label="Response">

  ### Response Body

  ```json
  {
    "fullName": "string",
    "username": "string",
    "email": "[EMAIL]",
    "mobile": "string",
    "forcePasswordChange": true,
    "roles": [...],
    "id": 0,
    "createdAt": "...",
    "updatedAt": "..."
  }
  ```

  </TabItem>
</Tabs>

<Callout type="warning" title="Response Note">

The response body can be optimized. It currently includes sensitive data like passwords.

</Callout>

## 2. Authenticate

Log in and receive access and refresh tokens.

### Environment Variables

- `IAM_JWT_ACCESS_TOKEN_TTL`: TTL for access tokens (default: 60 mins).
- `IAM_JWT_REFRESH_TOKEN_TTL`: TTL for refresh tokens (default: 1 day).

<Tabs defaultValue="request">
  <TabItem value="request" label="Request">

  ### Headers

  ```http
  Content-Type: application/json
  ```

  ### Request Body

  ```json
  {
    "email": "[EMAIL]",
    "username": "[EMAIL]",
    "password": "[PASSWORD]"
  }
  ```

  </TabItem>
  <TabItem value="response" label="Response">

  ### Response Body

  ```json
  {
    "data": {
      "user": {
        "email": "[EMAIL]",
        "username": "[EMAIL]",
        "roles": ["Admin", "Internal User"]
      },
      "accessToken": "<ACCESS_TOKEN>",
      "refreshToken": "<REFRESH_TOKEN>"
    }
  }
  ```

  </TabItem>
</Tabs>

## 3. Refresh Tokens

## 4. Forgot Password

Initiates and confirms password reset flow.

### Environment Variables

- `IAM_OTP_EXPIRY`: OTP expiry time (default: 10 mins).

<Tabs defaultValue="initiate">
  <TabItem value="initiate" label="Initiate">

  ### Request

  ```http
  POST /api/iam/initiate/forgot-password
  ```

  ### Headers

  ```http
  Content-Type: application/json
  ```

  ### Request Body

  ```json
  {
    "email": "[EMAIL]",
    "username": "string"
  }
  ```

  ### Response Body

  ```json
  {
    "status": "success",
    "message": "Password reset token sent",
    "data": {
      "user": {
        "email": "[EMAIL]",
        "username": "string"
      }
    }
  }
  ```

  </TabItem>
  <TabItem value="confirm" label="Confirm">

  ### Request

  ```http
  POST /api/iam/confirm/forgot-password
  ```

  ### Request Body

  ```json
  {
    "username": "string",
    "email": "[EMAIL]",
    "verificationToken": "string",
    "password": "[NEW_PASSWORD]"
  }
  ```

  </TabItem>
</Tabs>

  ```json
      "email": "[EMAIL]",

  ```

  </TabItem>
</Tabs>
