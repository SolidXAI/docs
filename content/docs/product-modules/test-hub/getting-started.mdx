import { Bot, CalendarClock, CirclePlus, Gauge, ListChecks, PlayCircle } from 'lucide-react';

Testing Hub helps your team make sure customer journeys are working before a release and whenever you need confidence in the application.

## Follow these steps

<Cards>
  <Card icon={<CirclePlus />} title="Add an Application" href="/docs/module-marketplace/testing-hub/add-product">
    Add the application you want to test, including its addresses, labels, email notifications, and webhook.
  </Card>
  <Card icon={<Bot />} title="Author Scenarios" href="/docs/module-marketplace/testing-hub/author-scenarios">
    Describe customer journeys to the connected MCP AI assistant.
  </Card>
  <Card icon={<PlayCircle />} title="Run Tests" href="/docs/module-marketplace/testing-hub/run-tests">
    Run all scenarios or focus on one labelled group.
  </Card>
  <Card icon={<ListChecks />} title="Understand a Test Result" href="/docs/module-marketplace/testing-hub/understand-test-result">
    See what passed and what failed.
  </Card>
  <Card icon={<CalendarClock />} title="Scheduling and Automation" href="/docs/module-marketplace/testing-hub/scheduling-and-automation">
    Schedule tests and choose email notifications or webhook updates.
  </Card>
  <Card icon={<Gauge />} title="Dashboard" href="/docs/module-marketplace/testing-hub/dashboard">
    See the current health of customer journeys and support release decisions.
  </Card>
</Cards>

<Callout type="info" title="Before you begin">
  Testing Hub must be available in your SolidX application. If it is not visible, ask an administrator to install it from the [Module Marketplace](/docs/module-marketplace).
</Callout>
