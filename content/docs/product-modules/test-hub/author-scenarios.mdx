Test scenarios are created through the connected MCP AI assistant. You describe what a customer should be able to do, and the assistant creates the scenarios for that application.

## Create scenarios

1. Open your application in **Applications**.
2. Open the **Scenarios** tab and select **Author Scenarios**.
3. In the connected MCP AI assistant, describe the customer journeys you want to test.

For example: *“Create test scenarios for the Library Management application: a librarian can sign in, add a book, and find a book.”*

The assistant creates the test scenarios from your description. You only need to explain the customer journey and expected outcome; you do not need to describe technical steps.

As the assistant writes them, the scenarios appear in the **Scenarios** tab, each with its labels and step count. You can search and filter them by tag.

![The Scenarios tab listing customer-journey scenarios with tags and step counts](/img/test-hub/scenarios-list.png)

<Callout type="tip" title="Start with the journeys that matter most">
  Begin with a small set of journeys that customers or staff need every day. Add more scenarios as the application grows.
</Callout>
